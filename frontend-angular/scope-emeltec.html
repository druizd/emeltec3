A · Alarmas: compartidas, con visibilidad y edición restringida — Definido

Modelo de alarmas Listo para construirM · 2-3 días

Alarmas compartidas por sitio (evita duplicados), con selección de quién puede verlas; solo Admin y Gerente (+ SuperAdmin) editan. Aplica a ambos sistemas (cámaras de frío + agua). Diseño genérico, reutilizable en otros procesos.

Decisiones cerradas
  • Default: visible para todo el sitio, restringible a una lista de usuarios.
  • Editan: Admin + Gerente (+ SuperAdmin). El resto solo ve.
  • Alcance: cold-room y agua.

Trabajo
  • DB: campo de visibilidad (visible_para[] + flag "todo el sitio") en cold_room_alarm_rule y alertas. Migración.

  • Frontend: multi-select "quién puede ver"; ocultar editar a no-admin/gerente.

Principio de diseño

Modular y repetible. Visibilidad de alarmas, observaciones y gestión de usuarios se construyen de forma genérica (por sitio/rol, no atados a cámaras de frío) para reutilizarse en futuros procesos (riles, eléctrico, proceso, pasteurizador).
Pendiente de definir antes de construir #2, #3, #4: fuente de alerta y alcance del export (#2); soft-delete y reset de contraseña (#3); flujo actual vs bandeja (#4). La fase de Alarmas (A) y #1 ya están listas/definidas.
